# Document Behaviors

Update project docs in the same pass as behavior changes.

## Minimum Update Set

1. Update `BEHAVIORS.md` for interaction changes.
2. Update the relevant file in `docs/` if architecture, persistence, or workflow changed.
3. If the tracker or project workflow changed, also update `docs/LOCAL_PROJECT_SYSTEM.md`.

Keep the docs short, concrete, and user-visible.
